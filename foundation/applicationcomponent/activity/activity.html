アクティビティとは

  • UI を持ち1つの画面を表すコンポーネント。
  • フォアグラウンドで実行状態、バックグラウンドで停止状態のように、見える(触れる)などの状態によるライフサイクルを持つ。

アクティビティの構成要素

  • アクティビティ自体は「Java のクラス」で、そこに UI部品（これも Java クラス）を追加することで画面が作られる。
  • アクティビティを構成する UI 部品としては大きく 2 種類がある。
    1. ビュー
      テキスト、ボタン、画像などの UI として形状を持つ部品
    2. レイアウト
      UI としての形状を持たずビューをグループ化し表示位置を制御する部品 ※ 正確にはレイアウトもビュー。UI形状を持つものと区別したいために呼び方をビューとレイアウトとしている。

イメージとして Activity 内にビュー、レイアウトのインスタンスが階層構造で追加されている感じ。XML の DOM と同じと考えても良い。

		    Activity
		        Layout
		            View1
		            View2
		        Layout
		            View3

アクティビティへのUIの追加

  • アクティビティに UI を追加する方法は 2 つある。
    1. UI 部品の配置を定義したXMLファイルからビューのインスタンスを生成しアクティビティに追加。このXMLファイルを 『レイアウトファイル』 という。
    2. UI 部品の Java クラスを new によりインスタンスを生成し 1 つずつアクティビティに追加。

※ 両方とも例は後述
