フラグメントとは

  • Android 3.0 でタブレットなどの大型画面の UI デザインに対応するために、画面を構成するペインとなる UI コンポーネント として導入された。

    「ペイン」とは、画面を複数の表示領域に分割して表示した際の、分割した個々の領域のことをいう。

  • 簡単に理解するなら、画面のペインくらいの大きい塊として扱える UI 部品 と思っておけばいい。

    Developers を見ると「Fragment は、Activity でのユーザー インターフェースの挙動や部位を表すもの」とある。まったくイメージが湧かない・・・

  • これだけだとビューをいっぱい持っているカスタムビューと大差ない気がする。カスタムビューと大きく違うのはフラグメントにはカスタムビューには無い、ペインとして扱うのに便利な下記のような機能が備わっている。
    1. アクティビティのようなライフサイクル(onStart, onResume, ...)
      → 画面のライフサイクルに合わせた処理が行いやすい。
    2. アクティビティスタックのようなフラグメントを積めるスタック
      → フラグメントだけを切り替えたり、back キーで戻したりと画面遷移のようなことができる。
    3. フラグメント単位での破棄・再生成
      → メモリが足りなくなった場合に解放できる？
    4. フラグメント変更のトランザクション
      → 便利？むしろこのせいでフラグメントが難しいものになっている・・・
  • フラグメントは 3.0 以降のネイティブに入っているし、support-v4 でも提供されている。新しいバージョンでフラグメントに機能やメソッドが追加されていることから、サポートライブラリを使う方が良さそう。（以降、サポートライブラリのフラグメントを使う前提で説明していく）
    ネイティブ　　　→ android.app.Fragment
support-v4　　→ android.support.v4.app.Fragment
