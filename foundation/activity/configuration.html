コンフィグレーションとは

  • コンフィグレーションとはデバイス構成情報やユーザー設定のこと。具体的には
    • デバイス構成情報
      キーボード、デバイスの向き、SIM検出など
    • ユーザー設定
      フォントサイズ、ロケールなど
  • コンフィグレーションが変更されると アクティビティは再生成される。

コンフィグレーションの変更でもアクティビティを再起動させないためには

　Android マニフェストの 要素の「android:configChanges」で構成が変わってもアクティビティを再生成させないように各構成単位で指定できる。 これを設定すると指定した項目の構成が変更された際に下記のメソッドが呼ばれるようになる。 つまり構成変更を通知してあげるから、対処は自分でやってねということ。 

@Override
public void onConfigurationChanged (Configuration newConfig)

例) デバイスの向きが変わっても再起動させない

activity
android:name=.SampleActivity　
android:configChanges=orientation|screenLayout;

コンフィグレーション情報をプログラムで取得

  • Configuration クラスはコンフィグレーション情報を表すクラス。
  • Resources から Configuration クラスのインスタンスを取得しあとはフィールドで情報を持っているのでそれを取得するだけ。
